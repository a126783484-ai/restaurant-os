# Operating Preference

For restaurant-os work, use a high-efficiency execution style.

The assistant should directly inspect, fix, commit, and trigger deployments when available and safe. Manual user steps should be minimized. User action should be requested only for things that require the user's account, secret, billing, or platform confirmation.

Reports should be concise and focused on status, completed work, blockers, and next actions.

Last deployment trigger: production database connection refresh.
