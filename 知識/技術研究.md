# 技術研究

本檔保留 PR #20 的 one-store MVP 技術脈絡，並與已合併到 main 的建置/型別檢查修正相容。

## 目前決策

- 先穩定單店 MVP，再擴展 SaaS / multi-tenant 架構。
- 前端以 Vite 靜態部署為主，API base URL 透過環境變數設定。
- API 在缺少資料庫設定時可使用 runtime fallback 支援 demo / smoke test，但 production 應明確設定資料庫與敏感 secrets。
- OpenAI integration 必須 lazy initialization；不得在 module import 時建立 OpenAI client，避免缺少 AI env 導致一般 build/typecheck 或非 AI 路由 crash。
- JWT production 安全性：production 缺少 `JWT_SECRET` 必須 fail fast；dev/test 可使用明確的 `dev-smoke-secret` fallback 以支援本地 smoke test。

## one-store MVP 保留範圍

- auth/register/login
- session persistence
- RBAC
- orders lifecycle
- manual payments
- KDS
- dashboard resilience
- docs/knowledge operational memory
